Contributors Editorial

Specs for contributors’ contribution to Marketplace.

Contributors: those who are already engaged, or want to get involved in an editorial capacity. Contributors are not every user of the Marketplace. A lot of the activities are going to involve a bit of effort rather than one-tap-and-done.

Issues we are working and would like to get feedback on

  • Types of contributors | GitHub issue
  • Recruiting new users and engaging existing contributors (who already contribute to other projects) | GitHub issue
  • Content types that contributors can create | GitHub issue
  • Sharing and promoting materials | GitHub issue
  • Recognising contributors | GitHub issue
  • Mozilla Editorial vs. Contributors Editorial | GitHub issue
  • Map a contributor’s customer journey through the service | GitHub issue

Content plan for this specs (subject to change)

  • High-level overview
    • Situations
    • Business problems we’re solving
    • etc. – Phil has worked on a lot of these
  • Mid-level strategies
    • Customer journey map
    • A page that explains each step of the journey: description, user needs, user activities, etc.
    • Design principles
  • Low-level concepts
    • Ideas that apply to specific parts of the journey: interest, recruitment, first usage, sharing, promotion, support, future contributions, etc.
    • Implementation details for each idea: screen flow, paper mockups, prototypes, visual design
